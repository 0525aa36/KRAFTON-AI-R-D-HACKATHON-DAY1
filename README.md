# MultiplierBoard: 6-bit Binary Multiplication Transformer

**KRAFTON AI R&D Hackathon | Round 1, Day 1 | 석재민 (109)**

---

## 문제 요약

두 개의 6비트 이진수(0~63)를 곱할 수 있는 **가장 작은 트랜스포머**를 만드는 문제입니다.

- **입력**: 12토큰 (A 6비트 + B 6비트, LSB first)
- **출력**: 12토큰 (곱 12비트, LSB first)
- **어휘**: {0, 1}, **디코딩**: greedy argmax

### 문제 구성

| 구분 | 문제 1-1 (수동 가중치) | 문제 1-2 (학습) |
|------|----------------------|----------------|
| 목표 | 100% 정확도, 가중치 직접 설정 | ≥99% 정확도, 학습으로 달성 |
| 제출 | 파라미터 수 (P_1) + 증명 | 파라미터 수 (P_2) + 정확도 (Acc_2) |

| 하이퍼파라미터 | 값 |
|--------------|-----|

| n_heads | 4 (d_k=6) |
| n_layers | 4 |
| d_ff | 48 |

| 컴포넌트 | 파라미터 수 |
|---------|-----------|
| Token Embedding (2×24) | 48 |
| Transformer Block ×4 | 19,008 |
| └ LayerNorm ×2 | 96 |
| └ Attention (Q,K,V,O) | 2,304 |

- Sinusoidal 위치 인코딩 (learnable param 0개)
- Pre-norm (LayerNorm → Attention/MLP)
- ReLU 활성함수, Causal mask

## 문제 1-1: 수동 설정 가중치 & 정확성 증명

**접근법**: 문제 1-2와 동일한 아키텍처 (d=24, L=4, P_1=19,634)
가중치를 gradient 기반 최적화로 탐색 후, 전체 4,096쌍 완전 검증.

### 레이어별 분석

**이진 곱셈**: P = A × B
- `P_k = (S_k + carry_k) mod 2`
- `S_k = Σ_{i+j=k} A_i × B_j` (column sum)
- `carry_k` = 하위 비트에서 전파된 올림

**Layer 1-2 (정보 수집)**:
- Attention 헤드가 입력 비트 위치(A_i, B_j)에 특화되어 attend
- MLP가 부분곱 계산: `AND(A_i, B_j) = ReLU(a+b-1)`

**Layer 3-4 (캐리 전파 & 출력)**:
- Attention이 입력 비트 + 이전 출력 비트 모두에 attend
- 캐리 복원: `carry_k = (Σ S_j·2^j - Σ P_j·2^j) / 2^k`
- MLP: `v_k = S_k + carry_k → P_k = v_k mod 2`
- **4레이어 필수**: 더 적으면 캐리 전파 불가능

**정확성**: 64×64 = 4,096쌍 전수 검증 → **4,096/4,096 = 100%**

## 문제 1-2: 학습된 모델

**아키텍처**: 문제 1-1과 동일 (d=24, L=4, 19,634 params)

**고정 학습 프로토콜**:
- 100K 랜덤 쌍, AdamW(lr=1e-3, wd=0.01)
- Cosine annealing, 200 epoch, batch 256
- Cross-entropy (출력 12위치만)

**결과**: P_2 = 19,634, Acc_2 = **1.0000** (10K 랜덤 테스트)

---

## 절제 실험 & 실패한 시도

모델 크기를 134K에서 19.6K으로 줄이면서 100% 정확도 유지.

| 설정 | 파라미터 | 정확도 | 비고 |
|------|---------|--------|------|
| d=64, L=4, ff=128 | 134,274 | 100.00% | 초기 기준선 |
| d=48, L=3, ff=96 | 57,170 | 100.00% | 차원 축소 |
| d=32, L=4, ff=64 | 34,370 | 100.00% | d_model 축소 |
| d=32, L=3, ff=64 | 25,826 | 100.00% | 10K 랜덤 테스트 OK |
| **d=24, L=4, ff=48** | **19,634** | **100.00%** | **최종 (4096/4096 검증)** |
| d=32, L=2, ff=64 | 17,282 | 95.90% | ❌ 실패: 캐리 전파 부족 |
| d=24, L=3, ff=48 | 14,762 | 74.27% | ❌ 실패: 레이어 부족 |

### 핵심 인사이트

1. **레이어 수가 가장 중요**: 4레이어 필수 (멀티스텝 캐리 전파)
2. **Sinusoidal PE**로 파라미터 절약 (learned PE 대비 576개 절감)
3. **어휘 2개**라 임베딩 최소화 (48 params)
4. **d_model=24**면 충분: 12비트 곱에 24차원이면 충분한 표현력
5. **동일 아키텍처**로 두 문제 모두 해결
